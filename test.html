Mario Code Control System Test

Open the main game at index.html to test the code-based control system.

Test Cases to Try:

  1. Basic Movement: 
    marioAPI.move("right", 5);
marioAPI.move("left", 3);
  2. Jumping: 
    marioAPI.jump();
marioAPI.move("right", 2);
  3. Complex Sequence: 
    marioAPI.move("right", 3);
marioAPI.jump();
marioAPI.wait(5);
marioAPI.move("left", 2);
marioAPI.fireball();
  4. Ducking (if Mario is large): 
    marioAPI.duck();
marioAPI.wait(10);
marioAPI.stopDuck();
  5. Position Checking: 
    var pos = marioAPI.getPosition();
console.log("Mario position:", pos);
marioAPI.move("right", 5);

Expected Behavior:

  • Mario should move right for 5 steps, then left for 3 steps
  • Mario should jump when jump() is called
  • Mario should wait for the specified number of frames
  • Mario should shoot fireballs if he has fire power
  • Mario should duck if he's large
  • Commands should execute in sequence from the queue

Debugging:

Open the browser console (F12) to see any error messages or debug output.
